# Contributing

Thanks for considering a contribution. RackHead Security is early-stage and defensive by design.

## Good contributions

- Clear documentation improvements
- Safe synthetic examples
- Threat-model templates
- Research notes with sources
- Defensive checks and evaluators
- Small reproducible tests

## Not accepted

- Malware, credential theft, persistence, stealth, bypass tooling
- Exploit code against real third-party systems
- Unsourced security claims about named products
- Doxxing or disclosure of secrets

## Style

Be precise, cite sources, state limitations, and avoid hype. Use terms like early-stage, defensive, toy example, work in progress, and not a production guarantee where appropriate.
